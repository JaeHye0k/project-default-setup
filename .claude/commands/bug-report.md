# Bug Report Generator

사용자가 제공한 버그 정보를 바탕으로 `docs/prompts/` 디렉토리에 버그 리포트 문서를 작성합니다.

## 입력

$ARGUMENTS

## 작성 형식

다음 형식에 맞춰 버그 리포트를 작성하세요:

```markdown
## 현상

[로컬과 배포환경의 차이점을 포함하여 버그 현상을 명확하게 기술]

## 환경

- [서버 환경, OS, 프록시 설정 등 관련 인프라 정보]
- [네트워크 구성 정보]
- [사용 중인 서비스 정보]

## 에러 로그

\`\`\`bash
[실제 에러 로그 또는 에러 메시지]
\`\`\`

## 원인

[버그의 근본 원인 설명]

<aside>
💡

**[관련 개념/용어]란?**

[개념에 대한 간단한 설명]

</aside>

### [원인 분석 소제목]

1. [단계별 원인 분석]
2. [...]

## 해결책

### [해결 방법 제목]

[해결 방법 상세 설명]

\`\`\`bash
[해결을 위한 명령어 또는 코드]
\`\`\`

### 시도했으나 실패한 방법 (있는 경우)

#### [실패한 방법 제목]

[왜 실패했는지 설명]
```

## 지침

1. 파일명은 버그 내용을 요약한 snake_case로 작성 (예: `auth_timeout_bug.md`)
2. 현상은 "로컬에서는 X하는데, 배포환경에서는 Y" 형식으로 차이점을 명확히
3. 에러 로그는 핵심 부분만 포함
4. 원인 분석 시 관련 기술 개념은 `<aside>` 블록으로 설명 추가
5. 해결책은 구체적인 명령어나 코드와 함께 제공
6. 시도했으나 실패한 방법도 기록하여 같은 시행착오 방지
