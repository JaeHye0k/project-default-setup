# 아키텍처 다이어그램 생성

프로젝트 코드를 분석하여 Mermaid 문법으로 시각적인 아키텍처 다이어그램 문서를 생성하세요.

## 생성할 다이어그램 목록

### 1. 전체 시스템 아키텍처 (graph TB)
- UI Layer, Service Layer, Data Layer, External Services로 구분
- 각 레이어의 주요 컴포넌트 표시
- 컴포넌트 간 연결 관계 표시
- 색상으로 레이어 구분

### 2. 데이터 플로우 다이어그램 (flowchart LR)
- 사용자 인터페이스에서 시작하는 데이터 흐름
- 브릿지/API 통신 경로
- 네이티브 서비스 처리
- 데이터 저장소 및 외부 API 연동

### 3. 컴포넌트/위젯 계층 구조 (graph TD)
- 진입점(main)에서 시작하는 계층 구조
- 주요 위젯/컴포넌트 트리
- 서비스 의존성 표시

### 4. 주요 기능별 시퀀스 다이어그램 (sequenceDiagram)
프로젝트의 핵심 기능들에 대해 각각 시퀀스 다이어그램 생성:
- 인증/로그인 플로우
- 주요 데이터 CRUD 플로우
- 외부 서비스 연동 플로우
- 실시간 통신 플로우 (있는 경우)

### 5. 상태 관리 다이어그램 (stateDiagram-v2)
- 앱 초기화 상태
- 주요 상태 전이
- 사용자 액션에 따른 상태 변화

### 6. 서비스 의존성 그래프 (graph TB)
- 진입점에서 시작
- Service Locator / DI 컨테이너 구조
- 전역 서비스들
- 외부 서비스 연동
- 헬퍼/유틸리티

### 7. 데이터 관점: 정보 흐름도 (flowchart TB)
- Data Sources (외부 API, 사용자 입력 등)
- Data Processing Layer
- Data Storage (로컬 DB, 캐시 등)
- Data Consumers (UI, 분석 등)

### 8. 기능 관점: 주요 유스케이스 (graph LR)
- 프로젝트의 핵심 기능별 유스케이스
- 각 유스케이스의 단계별 흐름

### 9. 이벤트/메시지 통신 패턴 (flowchart LR)
- 이벤트 발행자 (Publishers)
- 이벤트 버스/메시지 큐
- 이벤트 구독자 (Subscribers)
- 주요 이벤트 타입 (classDiagram)

### 10. 컴포넌트 상호작용 매트릭스 (표)
- 주요 컴포넌트 간 상호작용 관계를 표로 정리
- 읽기/쓰기, 발행/구독, API 호출 등 관계 유형 표시

### 11. 성능 고려사항 다이어그램 (flowchart TD)
- 성능 최적화 포인트
- 잠재적 병목 지점
- 최적화와 병목 간의 완화 관계

## 다이어그램 스타일 가이드

### 색상 규칙
- UI 레이어: 파란색 계열 (#e1f5ff, #e3f2fd)
- 서비스 레이어: 보라색 계열 (#f0e1ff)
- 데이터 레이어: 녹색 계열 (#e1ffe1, #c8e6c9)
- 외부 서비스: 빨간색 계열 (#ffe1e1)
- 이벤트/메시지: 노란색 계열 (#ffeaa7, #fff4e1)

### Mermaid 문법 규칙
- subgraph로 논리적 그룹 구분
- style 지시자로 노드 색상 지정
- 적절한 화살표 유형 사용 (-->, -.->. ==>)
- 노드에 설명 텍스트 포함

## 문서 형식

1. 마크다운 형식으로 작성
2. 한국어로 작성
3. 목차 포함
4. 각 다이어그램에 간단한 설명 추가
5. 다이어그램 렌더링 가이드 포함 (GitHub, VSCode, mermaid.live 등)
6. 문서 하단에 버전 정보 및 관련 문서 링크

## 분석 방법

1. 프로젝트 구조 파악 (디렉토리, 주요 파일)
2. 진입점 분석 (main.dart, index.js 등)
3. 의존성 파일 분석 (pubspec.yaml, package.json 등)
4. 서비스/컴포넌트 파일 분석
5. 라우팅/네비게이션 구조 파악
6. 상태 관리 패턴 파악
7. API/외부 서비스 연동 분석
