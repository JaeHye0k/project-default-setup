# 온보딩 문서 생성

프로젝트를 철저히 분석하여 새로운 개발자가 빠르게 프로젝트를 이해할 수 있는 종합적인 온보딩 문서를 생성하세요.

## 분석 대상

다음 항목들을 분석하여 문서에 포함하세요:

### 1. 프로젝트 개요
- pubspec.yaml 또는 package.json 파일에서 프로젝트명, 버전, SDK 요구사항 파악
- 프로젝트의 주요 목적과 특징 설명
- 패키지명 및 식별자

### 2. 기술 스택
- 의존성 파일(pubspec.yaml, package.json 등) 분석
- 핵심 프레임워크와 라이브러리를 카테고리별로 정리:
  - 핵심 프레임워크
  - 백엔드 및 클라우드 서비스
  - HTTP 및 API 통신
  - 상태 관리 및 의존성 주입
  - 로컬 스토리지
  - 미디어 및 이미지 처리
  - UI 컴포넌트

### 3. 프로젝트 구조
- 소스 디렉토리 구조를 트리 형태로 표현
- 각 디렉토리와 주요 파일의 역할 설명

### 4. 아키텍처 및 디자인 패턴
- 사용된 아키텍처 패턴 분석 및 설명
- 상태 관리 방식
- 이벤트 기반 통신 구조 (있는 경우)
- 데이터 흐름

### 5. 핵심 컴포넌트 상세
주요 파일들을 분석하여 다음을 문서화:
- 파일별 주요 책임
- 중요한 함수/클래스 설명
- 코드 예제 포함
- 초기화 흐름

### 6. 시작하기 가이드
- 사전 요구사항
- 설치 단계 (bash 명령어 포함)
- 환경 설정 방법
- 외부 서비스 설정 (Firebase 등)

### 7. 개발 워크플로우
- 새로운 기능 추가 방법
- 주요 컴포넌트 확장 방법
- 빌드 및 배포 명령어

### 8. API 및 외부 서비스
- API 엔드포인트 정보
- HTTP 클라이언트 설정
- 외부 서비스 통합 정보

### 9. 트러블슈팅
- 자주 발생하는 문제와 해결책
- 유용한 디버깅 명령어
- 디버깅 팁

### 10. 주요 파일 참조
- 핵심 파일 목록 (라인 수, 설명 포함)
- 서비스 레이어 파일
- 유틸리티 파일

## 문서 형식

1. 마크다운 형식으로 작성
2. 한국어로 작성
3. 이모지를 적절히 사용하여 가독성 향상
4. 코드 블록에는 언어 지정 (dart, bash, javascript 등)
5. 표를 활용하여 정보를 구조화
6. 체크리스트 형식의 시작 가이드 포함
